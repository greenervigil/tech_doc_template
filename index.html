Introduction

JavaScript is a small, lightweight, cross-platform, object-oriented scripting language. JavaScript can be connected to the objects of its environment to provide programmatic control over them. JavaScript contains a standard library of objects, such as Array, Date, and Math, and a core set of language elements such as operators, control structures, and statements. Core JavaScript can be extended for a variety of purposes by supplementing it with additional objects.

  • Client-side JavaScript extends the core language by supplying objects to control a browser and its Document Object Model (DOM).
  • Client-side extensions allow an application to place elements on an HTML form and respond to user events such as mouse clicks, form input, and page navigation.
  • Server-side JavaScript extends the core language by supplying objects relevant to running JavaScript on a server.
  • Server-side extensions allow an application to communicate with a database, provide continuity of information from one invocation to another of the application, or perform file manipulations on a server.

JavaScript vs Java

JavaScript and Java are similar in some ways but fundamentally different in many others. Javascript was originally named LiveScript. The JavaScript language resembles Java, but does not have Java's static typing or strong type checking. JavaScript follows most of the Java expression syntax, naming conventions and basic control-flow constructs which was the reason why it was renamed from LiveScript to JavaScript.

It is not necessary syntactically to use a semicolon at the end of a code line.

Java is a compile-time system of classes built by declarations. JavaScript supports a runtime system based on a small number of data types representing numeric, Boolean, and string values. JavaScript has a prototype-based object model instead of the more common class-based object model. The prototype-based model provides dynamic inheritance. This allows for variations of what is inherited for individual objects. JavaScript also supports functions without any special declarative requirements. Functions can be properties of objects, executing as loosely typed methods.

JavaScript is a "free-form" language compared to Java. You do not have to declare all variables, classes, and methods. You do not have to be concerned with whether methods are public, private, or protected, and you do not have to implement interfaces. Variables, parameters, and function return types are not explicitly typed.

Variables

Variables are the link in code to values.

Variables must start with a letter, underscore (_), or dollar sign ($); subsequent characters can also be digits (0-9). Because JavaScript is case sensitive, letters include the characters "A" through "Z" (uppercase) and the characters "a" through "z" (lowercase).

There are four ways in which you can declare a variable.

With the keyword var. It should be noted this way should no longer e utilized.

With the keyword let.

With the keyword const. This decalaration is meant to prevent changes to the variable.

const PI = 3.14

Or globally. More on this in a bit.

Variable Scope

When you declare a variable outside of any function, it is called a global variable, because it is available to any other code in the current document. When you declare a variable within a function, it is called a local variable, because it is available only within that function.

Prior to ECMAScript 2015 JavaScript had no concept of block statement scope. A variable declared within a block is local to the function (or global scope) that the block resides within.

The following code will log 15, because the scope of x is the function (or global context) within which x is declared, not the block, which in this case is an if statement.

if (true) { var x = 15; } console.log(x); // 15

This behavior changes with the introduction of the ECMScript let keywork.

if (true) { let y = 15; } console.log(y); // ReferenceError: y is not defined
Global Variables

Global variables are properties of the global object. In web pages the global object is the window, so you can set and access global variables using the window.variable syntax.

window.variable

You can access global variables declared in one window or frame from another window or frame by specifying the window or frame name. For example, if a variable called username is declared in a document, you can refer to this variable from an iframe as parent.username.

parent.username

Values

You cannot explain variables without then addressing what they link to; values. There are two types of values:

Primitive Values

Primitive values are immutable. This means these cannot be changed.

  • Undefined (undefined): used for unintentionally missing values.
  • Null (null): used for intentionally missing values.
  • Booleans (true and false): used for logical operations.
  • Numbers (-100, 3.14, and others): used for math calculations.
  • Strings ("hello", "abracadabra"): used for text.
  • Symbols (uncommon): used to hide implementation details.
  • BigInts (new): used for math on big numbers.

Objects and Functions

Objects and functions are changeable or mutable.

  • Objects ({}): used to group related data and code.
  • Functions (x => x * 2): used to refer to code.

Everything else in javaScript is nothing more than an object

Equality of Values

All values are considered "truthy" except the following exceptions:

  • null
  • undefined
  • NaN
  • empty string ("")
  • 0
  • false

What this means is you can easily use a value instead of evaluating the value.

let value = 'some text' if (value) { }

Types of equality

  • Strict Equality: a === b (triple equals).
  • Loose Equality: a == b (double equals).
  • Same Value Equality: Object.is(a, b) (are a and b the same value).
